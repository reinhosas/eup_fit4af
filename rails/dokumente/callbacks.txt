Die Entwickler von Rails sind bemüht Möglichkeiten zu schaffen, damit Code nicht wiederholt werden muss.

Beispiel:
Im Controller gibt es in vielen Actions die Notwendigkeit einen
bestimmten Event in der Datenbank zu finden und in der @event-variable zu speichern.
Damit diese Zeile Code nicht ständig widerholt werden muss kann sie in eine before_action asgelagert werden.
Wie der Name schon sagt wird diese Methode vor jeder anderen Action innerhalb des Controllers ausgeführt.
before_action :set_event, only: [:show, :edit, :update, :destroy]

Die Methode before_action erwartet als ersten Parameter den Namen einer Methode, die entweder im application_controller oder im
Controller selbst definiert ist. Alle weiteren Parameter sind optional.

Am Ende des event_controllers wird nun die Methode definiert:
	def set_event
		@event = Event.find(params[:id])
	end

Da diese Methode nicht außerhalb des Controllers aufgerufen werden soll, wird sie inerhalb eines als private deklarierten Bereichs
definiert. Alles was unterhalb von private steht ist von außen nicht erreichbar.
